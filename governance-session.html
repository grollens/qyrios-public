Agent Factory

5 principer för agentbyggande

1. Bygg fritt — dela medvetet

Alla får experimentera utan att fråga. Men: det som delas kräver synk med Simon, det som går i produktion kräver Samuels prioritering.

2. Enkel, ägd, spårbar

Det enklaste som fungerar vinner. Varje agent har en ägare (en person, inte "teamet"). Schemalagda agenter registreras, modellversion pinnas, ändringar loggas.

3. Läs fritt — skriv varsamt

Agenter som läser data: inga hinder. Agenter som skriver: kräver synk med Simon och tydliga guardrails. En människa äger alltid beslutspunkten.

4. Skydda data — döda det oanvända

Agenter ser aldrig mer data än de behöver. En agent som ingen använder ska avvecklas. Avveckling är lika viktigt som skapande.

5. Lär av varje körning

Varje agent ska ha en feedback-loop. Vi delar promptar och lärdomar aktivt. Evals och Langfuse-data är grundmaterialet — inte nice-to-have.

Agent Factory

Hur teamet arbetar med Claude — 8 byggstenar

Varje teammedlem har en personlig AI-miljö. Så här hänger delarna ihop.

1. Sessionsminne

Claude minns kontexten under en session. Varje konversation kan referera tillbaka till vad som sagts. Sessioner lever i chatten.

2. Skills (återanvändbara instruktioner)

Färdiga "recept" som Claude följer. Kan skapas av vem som helst. Aktiveras med /skill-namn. Tex: /skapa-produkt, /analysera-rapport.

3. MCP-kopplingar (PIM, Supabase, etc.)

Claude kan prata direkt med system via MCP. R1 gäller: read fritt, write kräver synk. Varje koppling = en connector.

4. Gemensamma skills (GitHub-repo)

Skills som delas via ett gemensamt repo. Alla kan använda, men ändringar kräver change-logg (R5). Versionshantering inbyggd.

5. Egna filkataloger

Varje användare kan peka Claude mot egna filer och mappar. Dokument, Excel, rapporter — Claude läser och arbetar med dem direkt.

6. Gemensam Gaston-kunskap

En delad kunskapsbas: processer, regler, produktinfo, personas. Samma "sanning" för alla. Fråga: vem äger och uppdaterar den?

7. Användarminne (Claudes minne)

Claude lär sig om dig över tid: preferenser, stil, vanliga frågor. Privat per användare. Byggs upp automatiskt.

8. Användarinstruktioner

Specifika regler per användare: "svara alltid på svenska", "använd vår tonalitet", "formatera som tabell". Sätts en gång, gäller alla sessioner.
